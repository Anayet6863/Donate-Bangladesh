What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.With the Document Object Model(DOM), programmers can create and build documents, navigate their structure, and add, modify, or delete elements and content. Anything found in an HTML or XML document can be accessed, changed, deleted, or added using the Document Object Model, with a few exceptions - in particular, the DOM interfaces for the internal subset and external subset have not yet been specified.

How do you select an element from the DOM?

In JavaScript, you can select elements from the DOM using several methods depending on what you're targeting:

  • getElementById() - Selects an element by its ID.
  • getElementsByClassName() - Selects all elements with a specific class name.
  • getElementsByTagName() - Selects all elements with a specific tag name.
  • querySelector() - Selects the first element that matches a CSS selector.
  • querySelectorAll() - Selects all elements that match a CSS selector.

    • What is event delegation in the context of the DOM, and why is it useful?

    Event delegation is a technique in the DOM where you attach a single event listener to a parent element, rather than to multiple child elements. The event listener listens for events triggered by child elements by taking advantage of event bubbling, where an event propagates from the target element up through the DOM hierarchy.
    Usefullness:
    Performance Optimization: Instead of adding multiple event listeners to several child elements, you only add one listener to a parent, reducing memory usage and improving performance, especially when there are many child elements. Handling Dynamic Elements: It allows you to manage events on dynamically added child elements. Since the event listener is attached to the parent, it automatically applies to any new child elements added after the listener is set. Cleaner Code: Centralizes event handling, making the code more maintainable and reducing redundancy.

    How do you manipulate an element's attributes and styles using the DOM?

    1. Manipulating Attributes:

  • getAttribute(): Retrieves the value of a specified attribute.
  • setAttribute(): Sets or updates the value of an attribute.
  • removeAttribute(): Removes a specified attribute from an element.
  • hasAttribute(): Checks if an element has a specific attribute.

    • 2. Manipulating Styles:
    Inline Styles:
  • style.property: Directly modify a specific CSS property using the style object.
  • style.cssText: Set multiple styles at once using a string of CSS code.
    • CSS Classes:
  • classList.add(): Adds a CSS class to the element.
  • classList.remove(): Removes a specific CSS class.
  • classList.toggle(): Toggles a class on or off.
  • classList.contains(): Checks if an element contains a specific class.
